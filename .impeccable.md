## Design Context

### Users
Organic search visitors needing quick file conversions (DOCX to Markdown, DjVu to PDF, image format changes, etc.). They arrive from Google, want to convert a file fast, and leave. Context: task-focused, low patience, comparing against other free tools.

### Brand Personality
**Trustworthy, efficient, friendly.** The interface should feel like a reliable tool that respects the user's time — warm but not frivolous, professional but not cold.

### Emotional Goals
- **Confidence & trust**: "This will work correctly and my file is safe"
- **Speed & simplicity**: "I can get this done in seconds, no friction"

### Aesthetic Direction
- **Tone**: Refined minimal with bold color accents per conversion format
- **Theme**: Light mode, warm off-white background, tinted neutrals
- **Typography**: Source Serif 4 (headings) + Instrument Sans (body)
- **Color system**: Teal primary (`oklch(0.45 0.08 160)`), per-format accent colors (blue, purple, red, amber, teal, orange, green, sky, emerald, violet) used on conversion cards and UI elements
- **Anti-references**: Avoid generic template look, AI slop aesthetics, default shadcn gray-on-gray

### Design Principles
1. **Format colors are identity** — each conversion type has a distinct accent color; use it consistently across cards, dropdowns, and interactive states
2. **Earn the click** — every interactive element should feel intentional and responsive (cursor-pointer, colored hover states, smooth transitions)
3. **Warm neutrals, not gray** — tint all neutrals toward the brand hue; avoid pure gray or pure black/white
4. **Function first, then delight** — the primary job is fast file conversion; design should clarify the path, not decorate it
5. **Designed, not templated** — avoid same-sized card grids, generic drop shadows, and default component library aesthetics
